继承父类其构造函数必须有 super #

class Person {
    constructor(public name: string, public ceshi: string) {}
}
class Teacher extends Person {
    constructor() {
        // 就是是父类没有构造函数，子类也要使用super()进行调用，否则就会报错。
        super('34', '34553'); // 由于父类的构造函数需要传递两个参数, 没有super将会报错
    }
}

const person = new Person('jspang', 'drfe')
console.log(person.ceshi)

泛型 #

使用场景 #

但现在有这样一个需求，就是 first 参数如果传的是字符串类型，要求 second 也传字符串类型. 同理，如果是 number 类型，就都是 number 类型。(意思就是第二个要用第一个的类型)

function join(first: string | number, second: string | number) {
  return `${first}${second}`;
}
join("jspang", ".com");

定义 #

泛型的定义使用 <> （尖角号）进行定义的

理解 #

泛型就是规定传进去的格式, 但是不指定它的类型, 同一个泛型的类型是一样的, 最好的方法就是看上面那个例子

泛型在数组中的使用 #

function myFun<T, P>(name: T, age: P): T {
  // 最后一个T用于定义函数的返回值, 前面的是定义函数的接受参数的类型
  return name;
}
let res = myFun<number, string>(455, "455");
console.log(res);

泛型简化基本类 #

class FindMax {
  constructor(private age: number) {}
  getAge(number: number) {
    return (this.age = number);
  }
}
let findmax = new FindMax(35);
console.log("findmax", findmax.getAge(0));

class FindMax<T> {
  constructor(private age: T) {}
  getAge(number: T): T {
    return (this.age = number);
  }
}
let findmax = new FindMax(35);
console.log("findmax", findmax.getAge(0));

泛型小例子 - 用泛型校验返回数据类型中必须包含某字段 #

// 定义一个Girl接口, 表示girl必须要有name, 和age
interface Girl {
  name: string;
  age: number;
}
// 定义了一个函数, 定义泛型, 函数接受泛型, 并且返回值也是泛型
function showGirl<T>(name: T): T {
  return name;
}

// 定义了一个函数, 定义泛型, 函数接受泛型, 返回值是继承Girl接口, 必须要有name, 和age
function showGirl<T, P extends Girl>(name: P): P {
  return name;
}

showGirl({ name: "34", age: 34 });

泛型的约束 #

// 这里的泛型必须是 number 或者 string 类型
function showGirl<T extends number | string>(name: T): T {
  return name;
}

showGirl(34); // 可以number 也可以string

ts 优缺点 #

优点: #

  1. TypeScript 增加了代码的可读性和可维护性
  2. TypeScript 非常包容
  3. TypeScript 拥有活跃的社区

缺点 #

  1. 有一定的学习成本，需要理解接口（Interfaces）、泛型（Generics）、类（Classes）、枚举类型（Enums）等前端工程师可能不是很熟悉的概念
  2. 短期可能会增加一些开发成本，毕竟要多写一些类型的定义，不过对于一个需要长期维护的项目，TypeScript 能够减少其维护成本
  3. 可能和一些库结合的不是很完美
