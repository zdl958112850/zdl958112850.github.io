vue2 的响应式 vue3 #

通过 Object.defineProperty 这个方法来实现的, 会把对象里面存在的属性进行读取和更改, 然后进行劫持, 问题: 咋们对目标对象的属性值读取和修改会分别进入 get 和 set 方法, 但是我们对新增的属性或者删除某个属性的时候, 界面是不会更新的;

对于数组: 数组：通过重写数组更新数组一系列更新元素的方法来实现元素修改的劫持 在 vue2 中想要把普通数组转为响应式数组, 要对数组中的原数据进行修改的话, 需要把更改原数组的更改方法进行重写操作, 从而实现元素的修改和劫持, 问题: 当直接通过下标替换元素或者更新 length 的时候也没不会发生改变, 比如 arr[1] = {} vue2 中就引入了 set 方法来实现

vue3 中实现方式: 通过 proxy 代理对象和 reflect(反射)来实现数据响应式代理的操作 传入两个参数: target 和 handler, target 就是目标对象, handler 就是一个处理器对象, 里面有用于监视数据的方法;

reflect 反射对象, 可以把目标对象相关的属性和属性值反射过来, 是一个静态方法不能 new, 通过直接调用里面的静态方法

vue3 的响应式数据的核心原理 #

const user = {
  name: 'zhangsan',
  age: 21,
  wife: {
    name: 'xiaoying',
    age: 18
  }
}

const proxyUser = new Proxy(user, {
  get(target, prop) {
    console.log('进入代理')
    return Reflect.get(target, prop)
  },
  // target 目标对象, prop参数名称, val更改的值
  // set对象不仅可以修改属性值, 还可以添加属性值
  set(target, prop, val) {
    console.log('进入set')
    return Reflect.set(target, prop, val)
  },
  // 删除目标对象的属性会进入
  deleteProperty(target, prop) {
    console.log('删除目标对象')
    return Reflect.deleteProperty(target, prop)
  }
})

const name = proxyUser.name
console.log('name', name)
proxyUser.age = 3432
console.log('user', user)
proxyUser.sex = '男'
console.log('proxyUser', proxyUser)
delete proxyUser.sex
console.log('proxyUser', proxyUser)
// 可以深度检测
delete proxyUser.wife
console.log('proxyUser', proxyUser)

ref 和 reactive 的细节问题 #

ref 也可以传入一个对象吗? 可以, 也会使用 reactive 进行处理, 页面使用的时候不用添加.value ref 内部：通过给 value 属性添加 getter/setter 来实现对数据的劫持 reactive 内部：通过使用 Proxy 来实现对对象内部所有数据的劫持，并通过 Reflect 操作对象内部数据 ref 的数据操作：在 js 中要.value, 在模板中不需要(内部解析模板时会自动添加.value)

// useClick.js
import {
    ref
} from "vue";

let x = ref(-1);
let y = ref(-1);

const clickFn = function(event: mouseEvent) {
    x.value = event.pageX;
    y.value = event.pageY;
}

onMounted(() => {
    window.addEventListener('click', clickFn)
})

onUnmounted(() => {
    window.removeEventListener('click', clickFn)
})

return {
    x,
    y
}

// App.vue

    const {
        x,
        y
    } = useClick();

    }
}

customRef 自定义 ref, 自定义 ref 的实现, 已达到延时操作 #

import {
    customRef
} from "vue";

function useDebouncedRef < T > (value: T, delay = 200) {
    let timeOutId: number

    return customRef((track, trigger) => {
        get() {
            track();
            return value
        }
        set(newValue: T) {
            clearTimeout(timeOutId);
            timeOutId = setTimeout(() => {
                value = newValue
                trigger();
            }, delay);
        }
    })
}
