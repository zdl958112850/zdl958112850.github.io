判断 object #

为什么 typeof null = "object"?

js 第一个版本遗留问题, 单个值在栈中占有 32 个存储单元, 分为两部分标记位 和数据, 标记位只有五种数据类型, null 是 0 到 31 位都是 0, 标记位也是 0,

为什么不修复? 兼容性问题, 处理了会带来更多问题

const arr = ['1', '2', '3'];

arr.map(parseInt)

function parseInt(string, 进制) {
    return 相应的进制的数
}
arr.map(function(item, index) {
    return item
})
// 结果: [1, NaN,  NaN], 将1按照0进制转,

typeof 和 instanceof #

typeof 和 instanceof 都是用来区分数据类型的

typeof 主要用于区分基本数据类型, instanceof 检测实例对象是否属于某个构造函数(从原型链上查找如果找到了就是他的实例)

需要注意:

  1. typeof null 为 object, typeof NaN 为 number, document.all() 返回 undefined
  2. typeof 用于区分复杂数据类型时, 除了 function 可以被输出 function, 其余都会输出 object
  3. typeof 后面可以跟未声明的变量
  4. typeof 会返回基本数据类型的字符串, instanceof 会返回 true 或者 false
  5. instanceof 精准判断复杂数据类型, 但不能精准判断基本数据类型

闲谈 #

  1. 百万级别 typeof 比 instanceof 性能高 2 倍
  2. null 是一个关键字, undefined 是一个变量

NaN #

isNaN:检查 toNumber 返回值，如果是 NaN,就返回 true,反 之为 false, 如果转数字出错, 就会抛出异常 Number.isNaN(): es6 新增,

如果没有 es6 环境, 可以通过这几种方法

Object.is(value, NaN); // 判断value是否等于NaN
value!==value // 利用NaN !== NaN 的特性

<!-- 综合 判断是否是NaN -->
if(!("isNaN" in Number)) {
    Number.isNaN = function(val) {
        return typeof val === "number" && isNaN(val)
    }
}

陷阱看本质 #

var arr = [NaN];
arr.indexOf(NaN); -1
arr.includes(NaN); true
// indexOf调用的是 Number的equal方法进行比对, equal方法在比较时, 看到NaN会返回false, 而includes用的sameValueZero进行比较, 会对NaN进行严格的比对

总结 #

  1. isNaN 是进行尝试转换, 然后把值跟 NaN 进行比价, Number.isNaN 没有转换, 然后直接进行值得比对
  2. indexOf 和 includes 的底层不一样
