前端开发方向 #

浏览器加载页面过程 #

下载 html, css, 下载完了后, html 通过 html parser 转为 DOM Tree, css 通过 css parser 转为 Style Rules, 最后附加在一起生成 Render Tree. 然后再通过布局引擎, 来对页面进行布局, 为什么需要 一个布局器呢? 因为浏览器宽度不一样, 会导致页面显示不同, 生成一个最终的 render tree. 那么 js 如何执行呢? js 引擎, 就是把 js 转为二进制的机器代码

webkit #

包含:

  • WebCore: 负责 HTML 解析、布局、渲染等等相关的工作
  • jsCore: 解析、执行 JavaScript 代码

V8 引擎的原理 #

v8 引擎原理-掘金

词法分析(逐字解析)和语法分析进行解析 通过语法分析转为 AST 树 (抽象语法树)

然后再由 ignition 库(转换器)转化为字节码 为什么不直接转为机器指令, 因为不知道在什么环境下运行, 类似 java 再由 v8 引擎转为不同平台的字节码

  • 由于从字节码再到机器码会浪费性能, 所以 v8 引擎会对于重复执行率较高的代码直接转为机器码, 只有到传的类型变化了的时候才会再次转为字节码, 最后转为机器码

  • V8 执行的细节 那么我们的 JavaScripti 源码是如何被解析(Parse 过程)的呢？ v8 引擎实现延迟解析, 将不必要的函数进行预解析, 也就是直解析需要的内容, 对于全量解析是在调用的时候进行

  • globalObject 也是 GO

  1. 代码被解析，v8 引/擎内部会帮助我们创建一个对象(Global0 bject->go)
  2. 运行代码 2.1 v8 为了执行代码，v8 引内部会有一个执行上下文栈（函数调用栈） executjion context stack 简称: ECStack

函数调用 #

this, 闭包, 作用域, GO, AO 如果是函数的话, 会开辟一片空间, 来存储函数, 该空间会包含父级作用域, 函数执行体, 会有一个内存地址, 全局里面保存的就是函数的内存地址; 执行函数的时候会创建一个 AO 对象, 执行完了会被销毁

  • 作用域链 包含 vo 和 父级作用域

  • 函数执行时候的操作

  • 函数遇到预解析的时候 会开辟一个空间, 会向 GO 里面存放该函数的内存地址, 调用栈,

  • 面试题

  var a = (b = 100);
}
foo();

  console.log("TCL: =======>", b);

  • 内存分配 栈空间 基本数据类型分配的地方, 保存复杂数据类型的引用 堆空间 分配复杂数据类型

  • 垃圾回收 GC 对于不需要使用的对象, 我们称之为垃圾, 他需要被回收, 释放空间

  • GC 算法 有一个引用计数, 当引用的时候, 会加 1, 引用被切断的时候会减一, 如果引用计数为 0 的时候, 就会清理

  • 总结: 函数执行的时候, 会向调用栈里面存放一个函数作用域, 一般情况下, 当该函数执行完毕之后, 该函数作用域就会被销毁

第七课 函数柯里化

什么是 currying #

只传递给函数一部分参数来调用它，让它返回一个函数去处理剩余的参数, 这个过程就称之为柯里化；

// 代码解释
// 例如:
function foo(m, n, x) {
  return m + n + x;
}
// 的一个函数转为下面函数的过程就是函数柯里化
function bar(m) {
  return function (n) {
    return function (x) {
      return m + n + x;
    };
  };
}
// 可以简化为:
bar = (m) => (n) => (x) => x + n + x;
bar(m)(n)(x);

作用 #

  1. 希望一个函数处理问题单一, 而不是将一大堆的处理过程交给函数来处理
  2. 逻辑的复用

封装 curry 函数 #

function hyCurry(fn) {
  function curried(...args) {
    // 如果传递的参数大于等于所需要的的参数, 直接执行
    if (args.length > fn.length) {
      return fn.apply(this, args);
    } else {
      return function (...args2) {
        return curried.apply(this, [...args, ...args2]);
      };
    }
  }
  return curried;
}

let 和 var 的区别

  1. let 具有块级作用域, var 具有函数作用域
  2. let 具有暂时性死区
  3. var 具有变量名提示
  4. var 对于一个变量可以重复定义
  5. let 对于一个变量不能重复定义
  6. 全局下面的 let 不会挂在到 windows 对象里面
